Исправление бага наград/прогресса шахтёра

Что было не так:
1. В runtime-config профессия шахтёра описана в секции profession.shahter,
   а код плагина для шахтёра использовал нормализованный ключ profession.rudy.
2. Из-за этого награды, сообщения level_up, level4_reward и часть настроек шахтёра
   читались не из той секции конфига.
3. Данные игроков при этом уже хранились в player-data.yml под ключом rudy,
   что видно по текущим данным сервера.

Что исправлено:
- Код теперь понимает алиасы профессии шахтёра: rudy <-> shahter.
- Для чтения настроек профессии используются fallback-ключи:
  * шахтёр: profession.rudy -> profession.shahter
  * рыбак: profession.fishman -> profession.fisher
- Исправлено чтение:
  * rewards.<level>.cmd
  * to_level.*
  * progress_caps.*
  * level_up.*
  * level_up_sound.*
  * level4_reward.*
- Нормализация ключей DataStore теперь также принимает shahter как алиас rudy.

Важно:
- Хранилище player-data.yml оставлено совместимым и продолжает использовать ключ rudy.
  Миграция данных не требуется.


Update 2:
- шахтер теперь сохраняется в player-data.yml под ключом shahter, а не rudy.
- старые данные rudy.* автоматически мигрируются в shahter.* при загрузке плагина.
- совместимость с командами/алиасами rudy сохранена, чтобы ничего не ломалось.
